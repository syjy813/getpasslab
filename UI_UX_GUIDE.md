## 0. 시작하기 전에 — 반드시 알아야 할 2가지

### 🔴 경고 1: Tailwind는 없다

정책 문서에는 "Tailwind CSS로 구현"이라고 적혀 있지만, **실제 코드에는 Tailwind가 없다.**
`package.json`에 tailwind 패키지가 존재하지 않으며, 스타일은 `src/styles/global.css` 단일 파일의 **순수 CSS + CSS Custom Properties**로 구현되어 있다.

```css
/* ❌ 동작하지 않음 */
<div class="bg-primary-600 rounded-lg px-6 py-4">

/* ✅ 실제 방식 */
<div class="subject-card">
/* global.css */
.subject-card {
  background: var(--gray-100);
  border-radius: var(--radius-lg);
  padding: var(--space-6);
}
```

> 이 문서는 **CSS 변수 표기를 정본**으로 삼는다. Tailwind 스케일은 참고용 대응표로만 병기한다.

### 🔴 경고 2: Lucide 패키지도 없다 — 하지만 Lucide는 쓰고 있다

`lucide` npm 패키지는 설치되어 있지 않다. 대신 **Lucide의 SVG path를 인라인으로 직접 복사**해서 쓰고 있다.

실제 GNB 코드:
```html
<svg width="14" height="14" viewBox="0 0 24 24" fill="none"
     stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round">
  <path d="m6 9 6 6 6-6"/>   <!-- ← Lucide의 chevron-down -->
</svg>
```

**이것은 올바른 판단이다.** 아이콘 8개를 쓰자고 npm 패키지 전체를 번들에 넣을 이유가 없다. 아이콘 스타일의 일관성은 **Lucide에서만 가져온다**는 규칙으로 유지된다.

## 1. 디자인 철학

### 1.1 한 줄 정의

> **"디자인이 콘텐츠를 방해하지 않는다."**

GetPassLab의 디자인은 **스스로를 드러내지 않는 것**을 목표로 한다. 사용자가 "이 사이트 예쁘다"고 느끼는 것보다, **"공부에 집중이 잘 된다"** 고 느끼는 것이 성공이다.

### 1.2 채택 방향: **미니멀·집중** (선택지 4개 중)

프로젝트 초기에 4가지 디자인 방향을 검토했다.

| 방향 | 인상 | 참고 사이트 | 판정 |
|------|------|-------------|------|
| A. 신뢰·전문성 | 정제, 학술적, 차분 | 노션, 토스(B2B) | 기각 |
| B. 친근·학습 친화 | 따뜻, 부드러움 | Quizlet, Duolingo | 기각 |
| **C. 미니멀·집중** | **깔끔, 콘텐츠 중심** | **Medium, Substack** | **✅ 채택** |
| D. 다이내믹·젊은 | 활기, 컬러풀 | 클래스101, Coupang Play | 기각 |

**C를 택한 5가지 근거 — 이것이 디자인 철학의 전부다**

| # | 근거 | 설명 |
|---|------|------|
| 1 | **콘텐츠가 핵심** | 디자인이 콘텐츠를 방해하면 안 된다 |
| 2 | **장시간 체류 → 시각적 피로 최소화** | 학습 사이트는 30분~2시간 머문다. 채도 높은 색·강한 대비는 피로를 누적시킨다 |
| 3 | **광고와의 조화** ★ | **디자인이 과하면 광고와 충돌한다.** 미니멀한 화면에서는 AdSense 배너가 오히려 자연스럽게 녹아든다. **수익 모델이 디자인 방향을 결정한 사례** |
| 4 | **개발 부담 최소화** | 단순한 디자인 = 빠른 구현. 1인 운영의 현실적 제약 |
| 5 | **콘텐츠 누적 시 자연스럽게 풍성해짐** | 출시 초기 74개 챕터로 시작해도 화면이 빈약해 보이지 않는다. 화려한 디자인은 콘텐츠가 적을 때 **더 초라해 보인다** |

> **③이 특히 중요하다.** 이 프로젝트에서 디자인은 미학적 선택이 아니라 **수익화 인프라**다. "광고가 튀지 않는 화면"이 곧 "광고 수익이 나오는 화면"이다.

### 1.3 시각적 레퍼런스: 토스(Toss) 인상 — 다만 **자산은 쓰지 않는다**

마일로가 직접 지정한 레퍼런스는 **토스 앱**이다.

**토스풍의 특징 (참고 대상)**

| 요소 | 특징 |
|------|------|
| 컬러 | 블루 액센트 + 모노톤 베이스 |
| 폰트 | 굵은 위계, 큰 헤드라인, 가독성 우선 |
| 라운드 | 강한 라운드, 카드·버튼 모두 둥글게 |
| 여백 | 넉넉한 화이트 스페이스 |
| 그림자 | **미세하거나 없음 (플랫에 가까움)** |
| 카드 | 카드 기반 레이아웃, 명확한 구분 |
| 위계 | 큰 텍스트 → 보조 텍스트 → 메타 정보 순 |

**⚖️ 라이선스 판단 (중요 — 법적 리스크 회피)**

토스 디자인 시스템(TDS)은 공개되어 있지만, **앱인토스(미니앱) 파트너사에게만 라이선스가 부여**된다. 자료의 권리는 지식재산권을 포함해 전부 토스에 귀속되며, 파트너사도 앱인토스 서비스 이용 범위 내에서만 사용 가능하다.

→ **GetPassLab 같은 외부 일반 웹사이트가 TDS 컴포넌트를 그대로 가져다 쓰는 것은 라이선스 위반이다.**

| ✅ 가능한 것 | ❌ 불가능한 것 |
|-------------|---------------|
| 토스의 **디자인 원칙·인상** 참고 | TDS 컴포넌트 코드 복사 |
| **컬러 톤·여백·라운드 감각** 차용 | TDS 피그마 라이브러리 사용 |
| 공개 블로그(toss.tech)의 사고방식 학습 | TDS 아이콘 라이브러리 |
| — | 토스 로고·브랜드 자산, Toss Product Sans 폰트 |

**최종 방침: "인상은 참고, 구현은 자체 디자인 토큰."**
마일로가 이를 명시적으로 승인했다: *"토스풍 인상 + 자체 디자인 토큰 정의 / 토스 블루 계열 차용 / 프리텐다드로 진행"*

### 1.4 암묵적으로 합의된 디자인 원칙 (문서에 없지만 결정된 것)

| # | 원칙 | 어디서 드러났나 |
|---|------|----------------|
| 1 | **PC 시안을 축소해서 모바일을 만들지 않는다** | Claude Design PC 시안이 나왔을 때, "레이아웃은 개발 시 새로 구성하고 **컬러·타이포·컴포넌트 스타일만 참고**"로 결정 |
| 2 | **라운드는 토스보다 정제한다** | 초기 토큰(6/10/14/20px)을 **4/8/10/14px로 하향 조정**. *"토스풍 라운드는 유지하되 조금 정제된 강도로"* — 학습 콘텐츠에는 과한 라운드가 가볍게 보인다는 판단 |
| 3 | **접기(accordion)를 남용하지 않는다** | 챕터 본문 5섹션은 **전부 펼침**. 접기는 ① SEO 불리 ② 학습 흐름 단절 |
| 4 | **다크모드는 하지 않는다 (지금은)** | MVP 제외. 단 토큰 체계가 있어 나중에 저비용 추가 가능 |
| 5 | **"준비 중"을 숨기지 않는다** | 빈 과목 카드를 숨기지 않고 "준비 중" 표시. 사이트가 성장 중임을 보여주는 것이 완성도 인상에 유리 |
| 6 | **운영자를 노출하지 않는다** | 푸터에 운영자 정보 없음. 개인 사이드 프로젝트가 아니라 **하나의 제품**으로 보이게 |
| 7 | **여백이 콘텐츠 부족을 가린다** | 자연 분량 원칙(1,000~1,300자)으로 본문이 짧아도, 넉넉한 여백과 큰 타이포로 빈약해 보이지 않게 |

## 2. 컬러 시스템

### 2.1 최종 팔레트 (v2 — 실제 구현값)

> ⚠️ **토큰이 2번 개정됐다.** 아래가 **최종본**이다. 초기안(§2.5)과 혼동하지 말 것.

#### 브랜드 컬러 (Primary)

| 토큰 | HEX | 용도 |
|------|-----|------|
| `--primary-50` | `#EFF6FF` | 매우 옅은 배경 (호버 영역, 선택 상태, 과목 태그 배경) |
| `--primary-100` | `#DBEAFE` | 옅은 배경 |
| `--primary-500` | `#2563EB` | 밝은 액센트 |
| **`--primary-600`** | **`#1D4ED8`** | **★ 메인 액센트** — 버튼, 링크, 강조, 활성 상태 |
| `--primary-700` | `#1E40AF` | 호버 상태 |
| `--primary-800` | `#1E3A8A` | 활성(눌림) 상태 |

> **`#1D4ED8`이 이 서비스의 정체성 컬러다.** 토스 블루의 인상은 유지하되 코드는 다르게 — 라이선스 안전지대.

#### 그레이스케일 (콘텐츠 베이스)

| 토큰 | HEX | 용도 |
|------|-----|------|
| `--gray-50` | `#F9FAFB` | 페이지 배경 (가장 옅음) |
| `--gray-100` | `#F2F4F6` | **카드 배경** |
| `--gray-200` | `#E5E8EB` | **구분선, 보더** |
| `--gray-400` | `#B0B8C1` | 비활성 텍스트, 플레이스홀더 |
| `--gray-600` | `#6B7684` | **보조 텍스트** (캡션, 메타 정보) |
| `--gray-800` | `#333D4B` | **본문 텍스트** |
| `--gray-900` | `#191F28` | **헤드라인** |

> **그레이 7단계가 화면의 90%를 차지한다.** 블루는 액센트로만 쓴다. 이것이 "모노톤 + 액센트 1색" 방향의 구현이다.

#### 의미 컬러 (Semantic)

| 토큰 | HEX | 용도 | GetPassLab 맥락 |
|------|-----|------|-----------------|
| `--success` | `#10B981` | 합격, 완료, 정답 | 정답 표시, 학습 완료 (Phase 2) |
| `--warning` | `#F59E0B` | 빈출, 함정 주의 | **`#빈출` 태그, "⚠️ 함정 주의" 인용블록** |
| `--error` | `#EF4444` | 오답, 위험, 경고 | 오답 표시 (Phase 2) |

### 2.2 컬러 사용 규칙

| 규칙 | 설명 |
|------|------|
| **본문 텍스트에 블루를 쓰지 않는다** | 링크만 예외. 본문은 `--gray-800` |
| **블루의 면적은 화면의 5% 이내** | 액센트는 희소해야 액센트다 |
| **채도 높은 색을 새로 만들지 않는다** | 위 팔레트 외의 색을 쓰지 않는다. 눈의 피로 |
| **하드코딩 금지** | `#1D4ED8`이 아니라 `var(--primary-600)` |
| **의미 컬러는 의미로만 쓴다** | `--warning`을 "예쁘니까" 장식에 쓰지 않는다 |

### 2.3 태그(Tag) 컬러 매핑

챕터 frontmatter의 `tags` 배열이 화면에 태그로 렌더된다. 태그 종류별 컬러:

| 태그 | 배경 | 텍스트 | 의미 |
|------|------|--------|------|
| **과목 번호** (`1과목`) | `--primary-50` | `--primary-600` | 소속 표시 |
| **`#빈출`** | `--warning` 계열 옅은 톤 | `--warning` 진한 톤 | **주목 유도 — 가장 중요한 태그** |
| **`#계산공식`** | `--gray-100` | `--gray-600` | 유형 분류 |
| **`#함정주의`** | `--warning` 계열 | `--warning` | 경고 |
| 기타 키워드 | `--gray-100` | `--gray-600` | 중립 |

> ⚠️ **정확한 태그별 컬러 매핑은 `global.css`에서 확인 필요.** 위는 정책 기반 재구성이다.
> 실제 코드에 `.hot` 클래스가 존재하는 것이 확인됐다 (`class={`tag ${... ? 'hot' : ''}`}`) — 빈출 태그 강조용으로 추정.

### 2.4 Tailwind 대응표 (참고용)

`#1D4ED8` = Tailwind `blue-700`, `#2563EB` = `blue-600`, `#EFF6FF` = `blue-50`.
즉 **브랜드 블루는 Tailwind 기본 팔레트 계열**이다. 반면 그레이는 **토스풍 커스텀 값**(`#191F28`, `#333D4B`, `#6B7684` 등)이다. 이 조합이 "토스 인상 + 안전한 브랜드 컬러"의 실체다.

### 2.5 폐기된 초기 팔레트 (v1) — 기록용

| 토큰 | v1 (폐기) | **v2 (최종)** |
|------|-----------|--------------|
| primary-500 | `#3182F6` | `#2563EB` |
| primary-600 | `#1B64DA` | **`#1D4ED8`** |
| primary-700 | `#1957C2` | `#1E40AF` |

**개정 이유**: 명시적 근거는 대화에 남아 있지 않다 ⚠️. 정황상 Claude Design 아카이브 제작 과정에서 표준 팔레트 계열로 정렬된 것으로 추정. **v1 값이 코드나 문서에서 발견되면 레거시다.**

## 3. 타이포그래피

### 3.1 폰트 패밀리

```css
font-family: 'Pretendard Variable', Pretendard, -apple-system,
             BlinkMacSystemFont, system-ui, Roboto, sans-serif;
```

**Pretendard 채택 근거**

| 근거 | 설명 |
|------|------|
| 한국어 최적화 | 한글 자소 균형이 우수. 국문 학습 콘텐츠의 장시간 가독성 |
| 오픈소스 · 무료 상업 이용 | 라이선스 리스크 0 |
| 한국 웹의 사실상 표준 | 사용자에게 낯설지 않음 |
| 토스풍 인상과 조화 | 레퍼런스와 시각적 정합 |

**기각안**: Toss Product Sans (라이선스 불확실), Inter + Noto Sans KR (영문·한글 분리의 복잡도)

**로드 방식** 🔴
현재 **jsDelivr CDN**으로 로드한다 (자체 호스팅 아님).
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable-dynamic-subset.min.css" />
```
> `dynamic-subset`을 쓰고 있다 — 페이지에 실제 등장하는 글자만 다운로드한다. **한글 폰트의 용량 문제를 해결하는 올바른 선택.**
> ⚠️ CDN 장애 시 시스템 폰트로 폴백된다. 자체 호스팅 전환은 저비용 개선 항목.

### 3.2 타입 스케일 (모바일 → 데스크톱)

| 토큰 | 모바일 | 데스크톱 | 굵기 | 용도 |
|------|--------|----------|------|------|
| `display` | 28px | 40px | Bold | Hero 카피 (홈) |
| `h1` | 24px | 32px | Bold | 페이지 제목, **챕터명** |
| `h2` | 20px | 24px | SemiBold | 섹션 제목 (`## 1. 핵심 공식 / 정의`) |
| `h3` | 18px | 20px | SemiBold | 챕터 헤더, 하위 섹션 |
| `h4` | 16px | 18px | SemiBold | **카드 제목** |
| `body-lg` | 16px | 17px | Regular | 본문 (중요 단락) |
| **`body`** | **15px** | **16px** | Regular | **본문 기본 ★** |
| `caption` | 13px | 14px | Regular | 보조 텍스트, 챕터 수, 한 줄 요약 |
| `label` | 12px | 13px | Medium | **태그, 뱃지, 회차 라벨** |

### 3.3 행간 (line-height) — 학습 가독성의 핵심

| 용도 | 값 | 근거 |
|------|-----|------|
| 헤드라인 | **1.3** | 타이트하게 → 덩어리로 인식 |
| **본문** | **1.6** | **학습 콘텐츠 가독성.** 한글은 영문보다 넉넉한 행간이 필요하다 |
| 캡션 | 1.5 | 중간값 |

> **본문 1.6은 협상 대상이 아니다.** 수험생이 30분간 읽는 텍스트다. 1.4~1.5로 줄이면 읽는 속도가 떨어지고 눈이 피로해진다.

### 3.4 타이포그래피 사용 규칙

| 규칙 | 이유 |
|------|------|
| **위계는 3단계 이내로** | 큰 텍스트 → 보조 텍스트 → 메타 정보. 4단계 이상은 위계가 무너진다 |
| **Regular / SemiBold / Bold 3종만 사용** | Light, Medium, ExtraBold 등을 늘리지 않는다 |
| **본문에 Bold를 남발하지 않는다** | 명사형 어미 문체에서 강조가 많으면 강조가 사라진다 |
| **명사형 어미 문체 전제** | 콘텐츠가 "~함", "~임", "~하는 구조"로 끝난다 → **문장이 짧고 스캔 가독성이 높다.** 타이포가 이 리듬을 지원해야 한다 |
| **수식은 KaTeX가 담당** | 본문 폰트와 수식 폰트가 다른 것은 정상이다. 억지로 맞추지 않는다 |

### 3.5 챕터 본문의 타이포 구조 (v3 템플릿 기준)

```
1과목 산업재해 예방…  [재해 발생 이론]   ← eyebrow: label / gray-600 + 태그
도수율                                  ← h1 / gray-900 / Bold
100만 근로시간당 재해 건수…              ← summary: body / gray-600
#계산공식  #빈출                        ← tags: label

## 1. 핵심 공식 / 정의                   ← h2 / gray-900 / SemiBold
본문…                                  ← body / gray-800 / lh 1.6
$$F = \frac{재해건수}{연근로시간수} × 10^6$$   ← KaTeX

> ⚠️ 함정 주의: …                        ← blockquote / warning 계열 배경

## 2. 단위·기호 해설                     ← h2
| 기호 | 명칭 | 단위 | 주의점 |            ← table / body / gray-800
```

## 4. 버튼 규칙

버튼은 **3종만 존재한다.** 4번째를 만들지 마라.

### 4.1 Primary — 메인 액션

| 속성 | 값 |
|------|-----|
| 배경 | `--primary-600` (`#1D4ED8`) |
| 텍스트 | 흰색, `body`, **SemiBold** |
| 패딩 | **M**: 세로 12px / 가로 20px<br>**L**: 세로 14px / 가로 24px |
| 라운드 | `--radius-md` (8px) |
| **호버** | 배경 `--primary-700` (`#1E40AF`) |
| **활성(눌림)** | 배경 `--primary-800` (`#1E3A8A`) |
| **비활성** | 배경 `--gray-200`, 텍스트 `--gray-400` |
| 전환 | `transition: background 150ms ease` |

**사용처**: "다음 챕터 →", **"필기 학습 시작 →"** (자격증 허브 CTA), **"1과목부터 시작 →"** (필기 페이지 CTA), 모달 확인

> **규칙: 한 화면에 Primary는 1개.** 두 개 이상이면 사용자가 무엇을 해야 할지 모른다.

### 4.2 Secondary — 보조 액션

| 속성 | 값 |
|------|-----|
| 배경 | 흰색 |
| 텍스트 | `--primary-600`, `body`, SemiBold |
| 보더 | 1px solid `--primary-600` |
| 라운드 | `--radius-md` (8px) |
| **호버** | 배경 `--primary-50` |
| 패딩 | Primary와 **동일** |

**사용처**: "← 이전 챕터", 보조 액션

### 4.3 Ghost — 텍스트 링크형

| 속성 | 값 |
|------|-----|
| 배경 | 투명 |
| 텍스트 | `--gray-800`, `body` |
| 보더 | 없음 |
| **호버** | 배경 `--gray-100` |
| 패딩 | 세로 8px / 가로 12px |

**사용처**: **GNB 메뉴**, 푸터 링크, 드롭다운 항목, 부가 액션

### 4.4 버튼 공통 규칙

| 규칙 | 근거 |
|------|------|
| **최소 터치 영역 44×44px** | 모바일 접근성 (§10) |
| **전환은 150ms** | 즉각적이면서 뚝 끊기지 않는 값 |
| **아이콘은 텍스트 오른쪽** | "다음 챕터 →" (화살표는 방향을 가리킨다) |
| **비활성 상태를 만들지 않는 것이 최선** | 누를 수 없는 버튼을 보여주는 것보다 안 보여주는 게 낫다. 예외: 실기 카드 "준비 중" |

## 5. 카드 규칙

### 5.1 베이스 스타일 — **3종이 동일 베이스를 공유한다**

**핵심 결정: 카드는 하나의 베이스만 있고, 콘텐츠만 다르다.** 카드 유형별로 스타일을 다르게 만들지 않는다.

| 속성 | 값 |
|------|-----|
| 배경 | `--gray-100` (`#F2F4F6`) 또는 흰색 |
| 라운드 | `--radius-lg` (**10px**) |
| 패딩 | 모바일 20px (`--space-5`) / 데스크톱 24px (`--space-6`) |
| 그림자 | `--shadow-sm` (기본) → 호버 시 `--shadow-md` |
| **보더** | **없음** (그림자로 구분한다) |
| **호버 효과** | 그림자 상승 + **`transform: translateY(-2px)`** |
| 전환 | `transition: all 200ms ease` |

> **보더 없이 그림자로만 구분하는 것이 토스풍 플랫 감각의 핵심이다.** 보더를 추가하면 화면이 답답해진다.

### 5.2 카드 3종

#### ① 과목 카드 (홈 · 필기 페이지)

```
┌────────────────────────────────┐
│ [1과목]                         │ ← label / primary-50 배경 / primary-600 텍스트 / radius-sm
│                                │
│ 산업재해 예방 및 안전보건교육      │ ← h4 / gray-900 / SemiBold
│                                │
│ 챕터 22개                       │ ← caption / gray-600
│                                │
│ #도수율 #하인리히 #위험성평가     │ ← caption / gray-600
└────────────────────────────────┘
```

| 영역 | 스타일 |
|------|--------|
| 과목 번호 태그 | `label` + `--primary-50` 배경 + `--primary-600` 텍스트 + `--radius-sm` |
| 과목명 | `h4`, `--gray-900`, 굵게 |
| 챕터 수 | `caption`, `--gray-600` — **`status: 작성완료`만 집계** |
| 대표 키워드 3개 | `caption`, `--gray-600`, `#` 접두 |

**예외 상태**: 챕터 0개 → **"준비 중" 표시** (카드를 숨기지 않는다)

#### ② 챕터 카드 (과목 목차 페이지)

```
┌────────────────────────────────┐
│ 도수율                          │ ← h4 / gray-900
│                                │
│ 100만 근로시간당 재해 건수        │ ← body / gray-800  (= frontmatter의 summary)
│                                │
│ [#계산공식] [#빈출]              │ ← label / 의미별 컬러
└────────────────────────────────┘
```

| 영역 | 스타일 |
|------|--------|
| 챕터명 | `h4`, `--gray-900` |
| 한 줄 요약 | `body`, `--gray-800` — **frontmatter `summary` 필드가 그대로 들어온다** |
| 태그 | `label`, **의미별 컬러** (`#계산공식` 중립 / `#빈출` warning / `#함정주의` warning) |

> **`summary` 필드는 3곳에서 쓰인다**: 카드 미리보기 + `<meta description>` + 챕터 헤더.
> **즉 `summary`는 UI 자산이자 SEO 자산이다.** 대충 쓰면 안 된다.

#### ③ 검색 결과 카드 (검색 모달)

```
┌────────────────────────────────┐
│ [1과목]  도수율                  │ ← 가로 배치, 컴팩트
│                                │
│ 100만 ==근로시간==당 재해 ...     │ ← 매칭 키워드 하이라이트
└────────────────────────────────┘
```

| 영역 | 스타일 |
|------|--------|
| 과목 태그 | inline, `caption` |
| 챕터명 | inline, `h4` |
| 스니펫 | `body`, `--gray-800` + **매칭 키워드에 `--primary-600` 하이라이트** |

**결과 표시 방식**: **평면 리스트 + 과목 태그** (과목별 그룹화 아님)
**빈 상태**: 인기 챕터 Top 5
**결과 없음**: 6과목 카드 6개

> ⚠️ **검색 모달은 미구현 상태다** (§0 경고 2 참조). 이 스펙은 설계 문서다.

### 5.3 카드 사용 규칙

| 규칙 | 근거 |
|------|------|
| **새로운 카드 유형을 만들지 않는다** | 베이스 재사용. 3종으로 모든 화면이 커버된다 |
| **카드 안에 카드를 넣지 않는다** | 중첩 그림자는 시각적 혼란 |
| **카드 그리드는 1→6 고정 순서** | 인기순·사용자 정렬 없음. **시험 과목 순서가 곧 학습 순서** |
| **호버 효과는 데스크톱만** | 모바일에는 hover가 없다. `@media (hover: hover)`로 감쌀 것 |

## 6. 아이콘 규칙

### 6.1 라이브러리: **Lucide** (인라인 SVG로 사용)

**채택 근거**

| 근거 | 설명 |
|------|------|
| 스타일 정합 | 깔끔·미니멀 → 토스풍 미니멀 감각과 일치 |
| 무료 · 상업 이용 가능 | 라이선스 리스크 0 |
| Astro 생태계 표준 | Vercel, Supabase 등에서 사용 |
| 필요 아이콘 전부 존재 | 약 500개 |

**기각안**: Heroicons (300개, Tailwind 팀 제작 — Tailwind를 안 쓰므로 이점 없음), Phosphor (800개, 두께 옵션 다양 — 과잉)

**구현 방식 🔴 (중요)**
`lucide` npm 패키지를 **설치하지 않는다.** Lucide 사이트에서 SVG를 복사해 **인라인으로 붙여 넣는다.**

```html
<!-- Lucide chevron-down — 실제 GNB 코드 -->
<svg width="14" height="14" viewBox="0 0 24 24" fill="none"
     stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round">
  <path d="m6 9 6 6 6-6"/>
</svg>
```

**근거**: 아이콘 8개를 쓰자고 npm 패키지를 번들에 넣을 이유가 없다. JS 번들 0을 유지한다.

### 6.2 사용 아이콘 목록

| 아이콘 | Lucide 이름 | 용도 |
|--------|-------------|------|
| 🔍 | `Search` | GNB 검색 → 검색 모달 (⚠️ **현재 GNB에 없음 — 미구현**) |
| ☰ | `Menu` | 모바일 햄버거 |
| ⌄ | `ChevronDown` | **드롭다운 (구현 확인됨)** |
| → | `ChevronRight` | 다음 챕터, 브레드크럼 구분 |
| ← | `ChevronLeft` | 이전 챕터 |
| ✕ | `X` | 모달 닫기 |
| ★ | `Star` | 즐겨찾기 (Phase 2) |
| 🏷️ | `Tag` | 태그 |
| ⚙️ | `Settings` | 설정 (Phase 2) |

### 6.3 아이콘 스타일 규칙

| 속성 | 값 |
|------|-----|
| 크기 | **20px** (기본, GNB 등) / **14~16px** (인라인, 텍스트 옆) / 24px (대형) |
| 색상 | **`stroke="currentColor"`** — 부모 텍스트 색을 상속. 하드코딩 금지 |
| `stroke-width` | **2** (Lucide 기본값. 바꾸지 말 것) |
| `stroke-linecap` / `linejoin` | **`round`** (Lucide 기본값) |
| `fill` | **`none`** (Lucide는 스트로크 기반 아이콘이다) |

| 규칙 | 근거 |
|------|------|
| **Lucide 외의 아이콘을 섞지 않는다** | 스타일 일관성이 무너진다. 이모지도 UI에 쓰지 않는다 |
| **`currentColor`를 반드시 쓴다** | 다크모드·호버 상태에서 자동으로 따라온다 |
| **아이콘만 있는 버튼에는 `aria-label` 필수** | §10 접근성 |
| **장식용 아이콘에는 `aria-hidden="true"`** | 스크린리더가 읽지 않도록 |

> ⚠️ **본문 콘텐츠의 이모지는 예외**: 챕터 본문의 "⚠️ 함정 주의" 인용블록에는 이모지를 쓴다. 이는 **UI가 아니라 콘텐츠**다.

## 7. 애니메이션

### 7.1 원칙: **거의 하지 않는다**

미니멀·집중 방향에서 애니메이션은 **주의를 분산시키는 비용**이다. 기능적 목적이 없는 애니메이션은 만들지 않는다.

### 7.2 허용된 애니메이션 (전부)

| 대상 | 효과 | 지속시간 | Easing |
|------|------|----------|--------|
| **버튼 호버** | 배경색 전환 | **150ms** | `ease` |
| **카드 호버** | `shadow-sm → shadow-md` + `translateY(-2px)` | **200ms** | `ease` |
| 드롭다운 열림 | (네이티브 `<details>` 기본 동작) | — | — |
| 모달 열림 | 페이드 인 | ⚠️ 미정의 | — |
| 페이지 전환 | View Transitions API | ⚠️ 미적용 추정 | — |

```css
/* 버튼 */
transition: background 150ms ease;

/* 카드 */
transition: all 200ms ease;
.card:hover { transform: translateY(-2px); box-shadow: var(--shadow-md); }
```

### 7.3 그림자 스케일 (애니메이션의 재료)

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.04)` | 카드 기본 |
| `--shadow-md` | `0 4px 12px rgba(0,0,0,0.06)` | 카드 호버, 드롭다운 |
| `--shadow-lg` | `0 8px 24px rgba(0,0,0,0.08)` | 모달, 검색 오버레이 |

> **불투명도가 0.04~0.08로 극히 낮다.** 토스풍 "플랫에 가까운" 감각을 유지하기 위함이다. 그림자를 진하게 만들지 마라.

### 7.4 금지되는 애니메이션

| 금지 | 이유 |
|------|------|
| 스크롤 기반 fade-in / parallax | 콘텐츠 읽기를 방해. **학습 사이트에서 최악의 패턴** |
| 로딩 스피너 | 정적 사이트라 로딩이 없다 |
| 자동 재생 캐러셀 | 주의 분산 |
| 바운스 / 이징 오버슈트 | "장난스러운" 인상 → 신뢰도 하락 |
| **광고 영역 애니메이션** | AdSense 정책 위반 소지 |

### 7.5 `prefers-reduced-motion` — **필수 구현** ✅

실제 `global.css`에 이미 구현되어 있다.

```css
@media (prefers-reduced-motion: reduce) {
  * { transition: none !important; }
}
```

> 전정기관 장애·편두통이 있는 사용자를 위한 OS 설정을 존중한다. **이미 구현되어 있으므로 새 애니메이션을 추가할 때 이 블록을 깨지 않도록 주의할 것.**

## 8. 반응형 기준

### 8.1 브레이크포인트

| 토큰 | 값 | 대상 |
|------|-----|------|
| `sm` | **640px** | 대형 스마트폰 |
| `md` | **768px** | 태블릿 |
| `lg` | **1024px** | 데스크톱 |
| `xl` | **1280px** | 와이드 데스크톱 |

### 8.2 **모바일 퍼스트** — 협상 불가

| 항목 | 내용 |
|------|------|
| **기본 스타일 = 모바일** | 미디어 쿼리는 `min-width`로만 확장 |
| **근거** | 수험생 트래픽 대부분이 모바일. 사용 맥락이 "문제 풀다 막혀서 폰으로 검색" |
| **금지** | **PC 시안을 축소해서 모바일을 만들지 않는다.** Claude Design PC 시안은 컬러·타이포·컴포넌트 스타일 참고용일 뿐, 레이아웃은 모바일부터 새로 짠다 |

### 8.3 컴포넌트별 반응형 동작

| 컴포넌트 | 모바일 (<768) | 태블릿 (768~1024) | 데스크톱 (>1024) |
|----------|---------------|-------------------|------------------|
| **GNB** | 로고 + [🔍] [☰] 햄버거 | 축약 가로 메뉴 | 로고 + 필기▼ + 검색창 |

## 9. 사용자 경험 원칙

### 9.1 사용 맥락 — 이것을 잊으면 전부 틀린다

> **수험생은 "공부하려고" 이 사이트에 오지 않는다. "막혀서" 온다.**

| 실제 사용 흐름 | UX 함의 |
|----------------|---------|
| 문제를 풀다 개념이 막힘 | **검색 유입이 대부분.** 홈을 거치지 않는다 |
| 모바일로 검색 | **랜딩 페이지 = 챕터 상세.** 홈이 아니다 |
| 짧게 확인 | **5~10분 완결.** 긴 글은 이탈 |
| 다시 문제로 복귀 | **이탈은 실패가 아니다.** 목적 달성 후 떠나는 것 |
| 한 세션에 여러 과목을 오간다 | **과목 간 횡이동이 1클릭이어야 한다** (과목 탭 바의 존재 이유) |

### 9.2 UX 원칙 7가지

#### ① 진입장벽 0
**로그인 없음. 회원가입 없음. 팝업 없음.** 광고 모델에서 진입장벽은 순손실이다.

#### ② 최단 경로로 콘텐츠에
- 필기 페이지의 과목 카드 → **과목 목차를 건너뛰고 첫 챕터로 직행**
- 자격증 허브 CTA "필기 학습 시작 →" → **1과목 첫 챕터로 직행**
- **결과**: 과목 목차 페이지의 진입 빈도가 떨어진다. **이 부작용을 알고도 감수했다** (챕터 사이드바에 "과목 전체 보기"를 두어 보완)

#### ③ 두 개의 진입 동선을 항상 제공
| 사용자 유형 | 동선 |
|-------------|------|
| 키워드를 아는 사용자 ("사망만인율") | **검색** |
| 과목 구조를 아는 사용자 | **6과목 카드** |
> 홈 Hero에 검색창과 과목 카드가 둘 다 있는 이유다.

#### ④ 현재 위치를 항상 알 수 있게
브레드크럼 + 사이드바 하이라이트 + 진행률 바("5 / 22") + 과목 탭 활성 표시.
**학습자는 "내가 어디쯤 왔나"를 알아야 계속한다.**

#### ⑤ 데이터로 말한다 — UI의 최우선 차별점
"5개년 4회 출제" 라벨, 회차 뱃지, "단독 출제 0건, 응용 결합형 100%" 같은 코멘트.
**이것이 이 사이트의 UI에서 가장 중요한 요소다.** 시각적으로도 눈에 띄게 배치해야 한다 (섹션 4).

#### ⑥ 이탈 대신 다음 행동을 제안
- 챕터 하단: 좌우 이동 버튼 + **다음 챕터명 미리보기** ("연천인율 →")
- 섹션 5: 관련 챕터 자동 링크
- 기출 모달: "이 챕터로 이동" 링크
> **PV를 늘리는 것이 곧 광고 수익이다.** 하지만 억지로 붙잡지 않는다 — 자연스러운 다음 단계만 제안한다.

#### ⑦ 미완성을 숨기지 않되, 노출하지도 않는다
- **미작성 챕터(스텁)**: 사용자에게 **완전히 보이지 않는다** (빌드에서 제외)
- **챕터 0개 과목**: **"준비 중" 표시** (숨기지 않는다)
- **실기 카드**: **"준비 중" 비활성** (숨기지 않는다)
> 차이: "만들다 만 것"은 감추고, "곧 나올 것"은 예고한다. 후자는 **재방문 동기**가 된다.

### 9.3 광고 UX 원칙

| 원칙 | 근거 |
|------|------|
| **저트래픽 페이지에는 광고를 넣지 않는다** | 자격증 허브 = 광고 0. UX만 해치고 수익 기여는 미미 |
| **검색 모달·정책 페이지에 광고 없음** | 짧은 확인 동선 / 법적 문서 |
| **챕터 상세에 3개 집중** | 수익의 80%+ 발생 페이지. 여기에 몰아준다 |
| **광고 슬롯은 고정 높이 예약** | CLS(레이아웃 시프트) 방지. 광고가 늦게 뜨면서 본문이 밀리면 최악의 UX |
| **광고가 튀지 않는 화면 = 미니멀 디자인** | §1.2의 근거 ③ |

## 10. 접근성 고려사항

> 접근성은 "장애인을 위한 배려"가 아니라 **품질의 지표**다. 접근성이 좋은 사이트는 SEO도, 모바일 UX도 좋다.

### 10.1 이미 구현된 것 ✅

| 항목 | 구현 |
|------|------|
| **`prefers-reduced-motion` 존중** | `global.css`에 구현 확인 |
| **네이티브 `<details>` 사용** | 키보드 조작·스크린리더·포커스 관리를 **브라우저가 공짜로 제공** |
| **시맨틱 HTML** | `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>` 사용 확인 |
| **`lang="ko"`** | `<html lang="ko">` 확인 |
| **본문 행간 1.6** | 난독증·저시력 사용자에게 유리 |
| **KaTeX 수식 (이미지 아님)** | 스크린리더가 읽을 수 있다. **이미지 수식이었다면 접근 불가** |

> **`<details>` 요소를 쓴 것이 접근성 측면에서 가장 큰 이득이다.** 커스텀 JS 드롭다운을 만들었다면 `aria-expanded`, `role="menu"`, 포커스 트랩, ESC 처리를 전부 직접 해야 했다.

### 10.2 반드시 지켜야 할 규칙

| # | 규칙 | 검증 방법 |
|---|------|-----------|
| 1 | **아이콘만 있는 버튼에 `aria-label`** | 검색 아이콘, 햄버거, 모달 닫기 버튼 |
| 2 | **장식용 아이콘에 `aria-hidden="true"`** | 드롭다운 chevron 등 |
| 3 | **포커스 표시(focus ring)를 제거하지 않는다** | `outline: none`만 쓰고 대체를 안 만드는 것이 최악 |
| 4 | **터치 영역 최소 44×44px** | 모바일 뱃지·태그가 작으면 누르기 어렵다 |
| 5 | **모달: ESC로 닫기 + 포커스 트랩 + 열릴 때 포커스 이동** | 기출문제 모달·검색 모달 |
| 6 | **모달 닫으면 원래 트리거로 포커스 복귀** | |
| 7 | **색만으로 정보를 전달하지 않는다** | `#빈출` 태그가 색만 다르면 색맹 사용자는 구분 못 함 → **텍스트 라벨 병기** |
| 8 | **`<table>`에 `<th>`와 `scope` 속성** | 챕터 섹션 2가 표다. 스크린리더가 셀-헤더 관계를 알아야 한다 |
| 9 | **이미지에 `alt`** | (현재 이미지가 거의 없으나, 기출 이미지 문제 도입 시 필수) |

### 10.3 명도 대비 (WCAG AA 기준: 본문 4.5:1, 대형 텍스트 3:1)

| 조합 | 대비비 (추정) | 판정 |
|------|--------------|------|
| `--gray-900` (#191F28) on 흰색 | 약 15:1 | ✅ 여유 |
| `--gray-800` (#333D4B) on 흰색 | 약 11:1 | ✅ 여유 |
| **`--gray-600` (#6B7684) on 흰색** | **약 4.6:1** | ✅ **간신히 통과** — 이보다 밝게 하지 말 것 |
| `--gray-600` on `--gray-100` (#F2F4F6) | 약 4.3:1 | ⚠️ **AA 미달 가능** — 카드 배경 위 보조 텍스트 주의 |
| 흰색 on `--primary-600` (#1D4ED8) | 약 7:1 | ✅ 여유 |
| **`--gray-400` (#B0B8C1) on 흰색** | **약 2.2:1** | ❌ **본문 사용 금지.** 비활성 상태·플레이스홀더 전용 |

> ⚠️ **검증 필요**: 위 대비비는 계산 추정치다. **실제 도구(WebAIM Contrast Checker)로 검증할 것.**
> 특히 **`--gray-600` 텍스트를 `--gray-100` 카드 배경 위에 올리는 조합**이 위험하다. 챕터 카드의 "한 줄 요약"·"챕터 수"가 여기에 해당한다.

### 10.4 알려진 접근성 리스크

| # | 리스크 | 심각도 |
|---|--------|--------|
| 1 | **기출 이미지 문제의 대체 텍스트 부재** | 🔴 이미지 자체가 없어서 **시각 장애 여부와 무관하게 모두가 문제를 이해할 수 없다** |
| 2 | **모바일 좌우 스와이프가 스크린리더와 충돌** | 🟠 스와이프 제스처는 스크린리더의 제스처와 겹칠 수 있다 |
| 3 | **카드 호버의 `translateY`가 전정기관 민감 사용자에게** | 🟡 `prefers-reduced-motion`으로 커버됨 (`transition: none`) |
| 4 | **KaTeX 수식의 스크린리더 읽기 품질** | 🟡 KaTeX는 MathML을 함께 출력하지만 읽기 품질은 브라우저마다 다름 |
| 5 | **광고가 키보드 탭 순서를 가로챔** | 🟠 AdSense iframe이 포커스 순서에 끼어들 수 있다. 도입 후 검증 |

### 12.4 브레이크포인트

| 토큰 | 값 |
|------|-----|
| `sm` | 640px |
| `md` | 768px |
| `lg` | 1024px |
| `xl` | 1280px |

### 12.5 실제 `global.css`에서 확인된 토큰명

> 정책서 토큰명과 실제 CSS 변수명이 다를 수 있다. **아래는 코드에서 직접 확인된 것들이다.**

```
--primary-600
--gray-900  --gray-600  --gray-200
--fs-caption  --fs-label
--fw-bold  --fw-semibold
--space-2  --space-3  --space-4  --space-6
--container-max
```

> ⚠️ **폰트 크기 토큰이 `--fs-*` 네이밍이다** (`--fs-caption`, `--fs-label`). 정책서의 `caption`/`label`과 대응된다.
> **`global.css` 전문을 읽고 이 문서의 §12를 실제 변수명으로 갱신할 것.**
