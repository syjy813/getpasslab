## 📅 ~2026-07-14 — Phase 4: 정리·이관 국면 시작

### `[결정]` 🎯 노션 동결 → git 단일 진실 원천
- **변경 내용**: 노션을 작업 도구에서 **읽기 전용 아카이브로 전환**. 진실 원천을 git으로 단일화
- **변경 이유**:
  1. 노션 플랜 제한으로 MCP 행 조회가 막힘
  2. 🚨 **이중 기입(노션 + md) → 장부-실물 불일치가 반복 발생** — Cowork가 챕터를 쓰면 노션 상태를 또 갱신해야 했고, 어긋나면 "장부엔 작성완료인데 실물은 없는" 상태가 됨
- **영향 범위**: **전체 데이터 아키텍처.** 콘텐츠 운영 워크플로 근본 변경
- **관련 기능**: F-33(`/admin/`), F-34(스텁 시스템), F-35~37(데이터 파이프라인)
- **근거**: `DECISION_LOG.md` D-68

### `[결정]` 챕터 메타데이터 → frontmatter 흡수 + 스텁 전량 생성
- **변경 내용**: 노션의 챕터 메타(`priority`/`status`/`order`)를 md frontmatter로 흡수. 미작성 챕터 171개를 **frontmatter만 있는 스텁 md**로 전량 생성
- **변경 이유**: **노션 의존을 완전히 소멸**시키고, 이후 양산을 "새 파일 생성"이 아닌 "스텁 채우기"로 전환
- **영향 범위**: `src/content/chapters/` 전체 (253개 파일 체계), `content.config.ts` 스키마 3필드 추가
- **관련 기능**: F-34
- **근거**: D-69, D-70

### `[결정]` 본문 DB화 기각
- **변경 내용**: 챕터 **메타데이터는 DB화(frontmatter)**, **본문은 DB화하지 않음**
- **변경 이유**: 본문은 LaTeX·표·인용이 섞인 장문 → 셀에 넣으면 관리 불능. **git이 줄 단위 이력을 제공하는 것이 본문 관리의 핵심 인프라.** DB화하면 "변환 계층"이 부활
- **영향 범위**: 데이터 아키텍처 원칙
- **관련 기능**: F-15(콘텐츠), F-37(스키마)
- **근거**: D-72

### `[결정]` "완벽 정리 후 이관"이 아니라 "이관 후 다듬기"
- **변경 내용**: 노션을 완벽히 정리한 뒤 이관하는 대신, 일단 이관하고 git에서 다듬기
- **변경 이유**: 어차피 이관하면 다듬는 작업이 git에서 훨씬 쉬움 (일괄 찾아바꾸기, diff, 롤백)
- **영향 범위**: 이관 작업 순서
- **근거**: D-74

### `[인프라]` 노션 CSV export 규칙 확립 (실패 경험)
- **변경 내용**: export 시 **반드시 `_all` 파일** 사용, **Include subpages 옵션 끄기**
- **변경 이유**: 기본 파일은 현재 뷰의 필터·숨긴 컬럼이 반영되어 행 누락 (실제로 "2과목 출시 필수" 뷰에서 12행만 export됨). subpages 켜면 md 1,690개 딸려옴
- **영향 범위**: 모든 향후 노션 export 작업
- **근거**: D-75
