## F-01. 홈 페이지

**기능명**: 홈 (`/`)

**목적**
사이트 전체의 진입점. 서비스 정체성을 1초 안에 전달하고, 사용자를 **가장 짧은 경로로 챕터 콘텐츠로 밀어넣는다.**

**사용자 가치**
- 처음 방문한 수험생이 "여기가 뭐 하는 곳인지"를 즉시 파악
- 검색어를 모르는 사용자는 과목 카드로, 키워드를 아는 사용자는 검색창으로 — **두 개의 진입 동선**을 동시에 제공

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | Hero 영역: 카피 + 검색창. 카피는 결합형 — "합격에 필요한 핵심만, 빠르게 / 산업안전기사 필기 6과목 요약" |
| 2 | 6과목 카드 그리드 (F-20) |
| 3 | 광고 #1 (6과목 카드 ↔ 인기 챕터 사이) |
| 4 | 인기 챕터 Top 10 (F-21) |
| 5 | 광고 #2 (인기 챕터 ↔ 최근 업데이트 사이) |
| 6 | 최근 업데이트 5개 (F-22) — 발행일 역순 |
| 7 | 과목 카드의 챕터 수 카운트는 **`status: 작성완료`만 집계** |

**예외사항**
- **챕터가 0개인 과목**: 카드를 숨기지 않고 **"준비 중" 표시**. (숨기면 사이트가 미완성으로 보이고, 표시하면 향후 추가 예고 효과)
- **인기 챕터 데이터가 없는 초기**: GA4 누적 전이므로 마일로 수동 선정값을 사용 (F-21 참조)

**화면 흐름**
```
홈 진입
 ├─ Hero 검색창 클릭 → 검색 모달 열림 (F-24)
 ├─ 6과목 카드 클릭 → 과목 목차 페이지 (F-04)
 ├─ 인기 챕터 클릭 → 챕터 상세 (F-05)
 └─ 최근 업데이트 클릭 → 챕터 상세 (F-05)
```

**정책**
- 과목 카드 정렬: **1→6 고정** (인기순·사용자 정렬 없음. 시험 과목 순서가 곧 학습 순서)
- 홈의 과목 카드 클릭 시 이동 대상은 **과목 목차 페이지**. (필기 페이지(F-03)의 과목 카드는 **첫 챕터로 직행** — 두 페이지의 카드 동작이 다르다. 의도된 차이)
- 광고 2개 배치

> ⚠️ **확실하지 않음**: 광고 수가 문서마다 1개/2개로 엇갈린다. 페이지 템플릿 문서에는 2개(6과목↔인기, 인기↔최근), 광고 총정리 표에는 1개로 기록되어 있다. 최종값 확인 필요.

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 인기 챕터를 GA4 데이터 기반 자동 산출로 전환
- 시험 D-day를 홈 Hero에도 노출하는 방안 검토 (현재는 자격증 허브에만)
- 자격증이 2개 이상 되는 시점에 홈 구조 전면 재설계 필요 (현재는 사실상 산업안전기사 전용 홈)

## F-02. 자격증 허브 페이지

**기능명**: 자격증 허브 (`/industrial-safety`)

**목적**
자격증 단위의 정보 허브. **자격증 확장(컴활2급 등)을 대비한 계층 구조의 존재 이유**이기도 하다.

**사용자 가치**
- 시험 일정, 응시자격, 합격기준 등 "공부 전에 알아야 할 것"을 한 곳에서 확인
- D-day 카운트다운으로 학습 긴박감 부여

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | Hero: "산업안전기사" + 한 줄 소개 |
| 2 | 자격증 기본 정보: **응시자격 / 시험구성 / 합격기준 / 시험 비용** |
| 3 | 시험 일정 표 + **D-day 카운트다운** (F-23) |
| 4 | 필기 / 실기 진입 카드 |
| 5 | CTA: **"필기 학습 시작 →"** (Primary 버튼, 1과목 첫 챕터로 직행) |

**예외사항**
- **실기 콘텐츠 미출시 상태**: 실기 카드를 **숨기지 않고 "준비 중" 비활성 처리**. 클릭 불가, 안내만 표시.
  - 근거: ① 향후 실기 추가를 예고 → 재방문 동기 ② 사이트 완성도 인상
- 시험 일정이 아직 발표되지 않은 연도: ⚠️ 처리 방식 미정의

**화면 흐름**
```
홈 → (GNB 또는 브레드크럼) → 자격증 허브
 ├─ "필기 학습 시작" CTA → 1과목 첫 챕터 (F-05)
 ├─ 필기 카드 → 시험구분 페이지 (F-03)
 └─ 실기 카드 → 비활성 (클릭 불가)
```

**정책**
- **합격률 통계 · 자격증 활용처: 의도적으로 제외.** 근거: 콘텐츠 유지보수 부담 대비 학습 동기 기여도가 낮고, 합격률은 수험생에게 오히려 위축 효과를 줄 수 있다고 판단
- **광고 미배치.** 근거: 트래픽이 낮은 페이지에 광고를 넣으면 UX만 해치고 수익 기여는 미미
- 시험 일정은 **수동 갱신** (Q-net API 미사용 — F-23의 폐기 사유 참조)

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 시험 일정 데이터를 별도 JSON/frontmatter로 분리해 매년 1회 갱신을 1파일 수정으로 처리
- 자격증 2개 이상 시점에 이 페이지가 자격증별 템플릿으로 일반화되어야 함
- **출시 직전 Q-net 공식 데이터 재확인 필수** (응시자격 / 합격기준 / 시험 비용 / 일정). 시험 비용은 필기 19,400원 / 실기 22,600원으로 기록돼 있으나 **연도별 변동 가능** ⚠️

## F-03. 시험구분(필기) 페이지

**기능명**: 필기 (`/industrial-safety/written`)

**목적**
필기 시험에 집중된 진입 허브. 6과목으로 분기시키는 것이 유일한 목적.

**사용자 가치**
- 필기 시험 형식(4지선다, 과목당 20문제)과 합격기준(평균 60점 이상, 과목별 40점 이상)을 확인
- 6과목 어디로든 1클릭 진입

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 브레드크럼: 홈 > 산업안전기사 > 필기 |
| 2 | 필기 시험 안내: 6과목 명단 / 시험 형식 / 합격기준 |
| 3 | CTA: **"1과목부터 시작 →"** (Primary) |
| 4 | 6과목 카드 그리드 (1→6 고정) — **카드 클릭 시 해당 과목의 첫 챕터로 직행** (과목 목차 페이지를 건너뜀) |
| 5 | 광고 #1 (6과목 ↔ 인기 챕터 사이) |
| 6 | 인기 챕터 Top 10 (필기 한정) |

**예외사항**
- **홈과의 콘텐츠 중복 우려**: 출시 초기에는 사이트 전체 = 필기이므로 인기 챕터 데이터가 홈과 동일하다. **이를 알고도 페이지를 유지**하기로 결정.
  - 근거: 실기 추가 후 자연스럽게 분리된다. **구조 일관성이 일시적 중복보다 중요.**
- 홈과의 차별화: 홈에는 있는 **"최근 업데이트" 섹션을 필기 페이지에서는 제외**

**화면 흐름**
```
자격증 허브 → 필기 카드 → 필기 페이지
 ├─ "1과목부터 시작" CTA → 1과목 첫 챕터
 ├─ 과목 카드 클릭 → 해당 과목 첫 챕터 (F-05) ★ 과목 목차를 건너뜀
 └─ 인기 챕터 클릭 → 챕터 상세
```

**정책**
- **과목 카드 클릭 = 첫 챕터 직행.** 이 결정의 부작용을 인지하고 있다: **과목 목차 페이지(F-04)의 진입 빈도가 떨어진다.** 이를 보완하기 위해 챕터 사이드바에 "과목 전체 보기" 메뉴를 두기로 함
- **광고 1개 포함** (자격증 허브와 달리 트래픽이 있는 페이지이므로)

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 실기 페이지(`/industrial-safety/practical`) 신설 시 이 템플릿을 재사용
- 인기 챕터를 "필기 한정" 필터링으로 실제 분리 (현재는 전체 = 필기라 동일)

## F-04. 과목 목차 페이지

**기능명**: 과목 목차 (`/industrial-safety/written/{과목-slug}`)

**목적**
한 과목의 전체 챕터를 그룹 단위로 조망. 학습 계획을 세우는 페이지.

**사용자 가치**
- 과목 안에 무엇이 있는지 전체 지도를 본다
- 그룹(중단원) 단위로 접었다 펼쳐 원하는 영역만 탐색

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 과목 헤더: 과목 번호 / 과목명 / 한 줄 소개 / 챕터 수 |
| 2 | 과목 내 검색창 + **정렬 옵션 3종 (기본 / 인기 / 가나다)** |
| 3 | 챕터 그룹 1~N: 그룹 헤더 + 챕터 카드. **그룹 기본 접힘**, 클릭 시 펼침 |
| 4 | 챕터 카드 구성: **챕터명 / 한 줄 요약 / 태그** |
| 5 | 광고 #1 (그룹 1↔2 사이), 광고 #2 (페이지 하단) |
| 6 | "이 과목 전체 보기" 버튼 → `/all` 페이지 (F-06) |
| 7 | 페이지 하단 6과목 미니 그리드 (다른 과목 이동) |
| 8 | 챕터 수 카운트 및 카드 표시는 **`status: 작성완료`만** |

**예외사항**
- **미작성(스텁) 챕터**: 카드에 표시하지 않는다. 카운트에도 넣지 않는다.
- **그룹 기본 접힘의 부작용**: 사용자가 아무것도 못 볼 수 있다 → ⚠️ 첫 번째 그룹은 기본 펼침으로 할지 미정의
- **챕터 카드 영문명**: 표시하지 않음 (슬러그는 URL에만 존재)

**화면 흐름**
```
홈/필기 페이지 → 과목 카드 → 과목 목차
 ├─ 그룹 헤더 클릭 → 펼침/접힘 토글
 ├─ 챕터 카드 클릭 → 챕터 상세 (F-05)
 ├─ 과목 내 검색 → 해당 과목 챕터만 필터링
 ├─ 정렬 토글 → 기본/인기/가나다 재정렬
 ├─ "전체 보기" → /all 페이지 (F-06)
 └─ 6과목 미니 그리드 → 다른 과목 목차
```

**정책**
- **과목 헤더에 "최종 업데이트" 표시 제외** (관리 부담 대비 가치 낮음)
- 이 페이지는 **필기 페이지에서 카드 클릭 시 건너뛰어진다** (F-03 정책). 진입 경로는 ① 홈 과목 카드 ② 챕터 사이드바의 "과목 전체 보기" ③ 6과목 미니 그리드 ④ 직접 URL

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 정렬 "인기" 옵션은 GA4 데이터가 쌓여야 실효성이 생김
- 그룹별 학습 진행률 표시 (Phase 2, 로그인 후)
- 미작성 챕터를 "준비 중" 회색 카드로 노출할지 재검토 (현재는 완전 비노출 — 사이트 규모가 작아 보이는 단점 존재)

## F-05. 챕터 상세 페이지 ★ 수익 핵심

**기능명**: 챕터 상세 (`/industrial-safety/written/{과목-slug}/{챕터-slug}/`)

**목적**
**서비스의 본체.** SEO 랜딩 페이지이자 광고 수익의 80% 이상이 발생하는 페이지.

**사용자 가치**
- 하나의 개념을 5~10분 안에 완결적으로 이해
- **이 개념이 실제로 몇 번 출제됐는지, 어떤 형태로 나왔는지**를 데이터로 확인 (경쟁 서비스에 없는 가치)

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 사이드바 (F-09): 같은 과목 챕터 목록 + 그룹 헤더 + 진행률 바 + 현재 챕터 하이라이트 |
| 2 | 브레드크럼 (F-08): 홈 > 산업안전기사 > 필기 > [1과목 ▼] > 도수율. 과목명 부분은 **드롭다운** |
| 3 | 챕터 헤더: 제목 / 진행률 / 태그 / "전체 스크롤로 보기" 토글 (→ `/all`) |
| 4 | **본문 5섹션** (F-15) — 전부 펼침 |
| 5 | 광고 3개: 섹션 3↔5 사이 + 페이지 하단 (수익 핵심) |
| 6 | 좌우 이동 버튼 + **미리보기 텍스트** ("← 강도율 / 연천인율 →") |
| 7 | 모바일 좌우 스와이프로 챕터 이동 (F-11) |
| 8 | `status: 작성완료`인 챕터만 페이지 생성 (`getStaticPaths` 필터) |

**예외사항**
- **미작성(스텁) 챕터**: 빌드에서 **페이지 자체를 생성하지 않는다.** dist에 새어나오면 이관 작업 실패로 간주
- **섹션이 비어 있는 경우**: 해당 섹션 **숨김** 처리
- **기출 매칭이 0건인 챕터**: 섹션 4(기출 출제 이력)를 어떻게 표시할지 ⚠️ 정의되지 않음 — "출제 이력 없음" 표시 vs 섹션 숨김
- **사이드바 표시 범위**: 작성완료 챕터만 (`sideChapters` 필터)

**화면 흐름**
```
과목 목차 / 검색 / 인기 챕터 → 챕터 상세
 ├─ 브레드크럼 과목 드롭다운 → 다른 과목으로 즉시 이동
 ├─ 사이드바 챕터 클릭 → 같은 과목 다른 챕터
 ├─ 섹션 4 회차 뱃지 클릭 → 기출문제 모달 (F-17)
 ├─ 섹션 5 관련 챕터 클릭 → 해당 챕터 (Soft Navigation)
 ├─ 좌/우 이동 버튼 (또는 모바일 스와이프) → 이전/다음 챕터
 └─ "전체 스크롤로 보기" → /all 페이지 (F-06)
```

**정책**

| 항목 | 값 | 근거 |
|------|-----|------|
| 본문 섹션 표시 | **5섹션 모두 펼침. 접기·탭 미사용** | 초기에는 "1·2·3 펼침 + 4~7 접기" 정책이었으나, 7섹션→5섹션 축소 후 **접기가 불필요**해짐. 접기는 SEO에도 불리 |
| 학습 보조 기능 | **미제공** | (형광펜, 메모 등) MVP 과스코프 |
| 공유 기능 | **미제공** | 수험생 사용 맥락에서 공유 니즈 낮음 |
| 광고 | **3개** | 서비스 내 최다. 수익의 80%+ 발생 페이지 |
| URL에 챕터 번호 미포함 | 확정 | **재정렬 자유 확보** — 챕터 순서가 바뀌어도 URL이 안 바뀜 |

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 기출 0건 챕터의 섹션 4 처리 정책 확정 (⚠️ 현재 미정의)
- 구조화 데이터(JSON-LD, Article/FAQPage 스키마) 삽입 → 검색 결과 리치 스니펫
- Phase 2: 챕터별 학습 완료 체크, 북마크 버튼

## F-06. 과목 전체 보기 (`/all`)

**기능명**: 과목 전체 스크롤 페이지 (`/industrial-safety/written/{과목-slug}/all`)

**목적**
한 과목의 모든 챕터를 **한 페이지에 스크롤형으로 일괄 노출**. "정독" 사용자를 위한 대체 뷰.

**사용자 가치**
- 시험 직전 과목 전체를 훑을 때 클릭 없이 스크롤만으로 통독
- URL 하나로 과목 전체를 공유·북마크

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 브레드크럼: 홈 > 산업안전기사 > 필기 > 1과목 > 전체 보기 |
| 2 | 과목 헤더 + "챕터별 보기로 전환" 버튼 (→ 첫 챕터 페이지) |
| 3 | **스크롤 진행률 바** (상단 sticky) |
| 4 | 그룹 헤더 → 챕터 콘텐츠 일괄 노출 (각 챕터 5섹션 전부 적용) |
| 5 | 챕터 사이 **명확한 divider + 챕터명 헤더** |
| 6 | **광고: 3챕터마다 1개** (일정 간격) |

**예외사항**
- 챕터가 많은 과목(4과목 48개)은 페이지가 매우 길어진다 → ⚠️ 총 광고 수 상한 미정의
- 미작성 챕터는 당연히 제외

**화면 흐름**
```
과목 목차 "전체 보기" 버튼  ─┐
챕터 헤더 "전체 스크롤로 보기" ─┴→ /all 페이지
 └─ "챕터별 보기로 전환" → 첫 챕터 페이지 (F-05)
```

**정책**
- **SEO: 챕터별 페이지와 별도로 인덱싱한다.** 중복 콘텐츠 페널티 우려가 있으나 감수. (⚠️ **재검토 필요** — canonical 태그로 챕터 페이지를 정본으로 지정하는 것이 안전할 수 있음)
- URL 공유 가능

**우선순위**: P2 (MVP 필수 아님)
**현재 구현 여부**: 🟡 **설계 완료, 구현 여부 미확인.** 구현된 5개 페이지 목록(홈·허브·필기·과목 목차·챕터 상세)에 `/all`이 없음 → **미구현 가능성 높음**

**앞으로 개선 사항**
- **중복 콘텐츠 SEO 리스크 재검토 (최우선)**. `rel=canonical`로 챕터 페이지를 정본 지정 검토
- 광고 총량 상한 정책 수립
- 목차 사이드바(챕터 점프) 추가

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**

## F-09. 챕터 사이드바

**우선순위**: P0

## F-10. 과목 탭 바

**기능명**: 과목 탭 바 (Subject Tab Bar)

**목적**
**필기 하위 어느 페이지에서든 과목 이동을 1클릭으로.**

**사용자 가치**
산업안전기사 수험생은 **한 학습 세션에 여러 과목을 오간다.** 현재 구조에서 과목 이동은 GNB 드롭다운 또는 브레드크럼 역행뿐이라 2~3클릭이 들고, 모바일에서는 챕터 사이드바가 숨어 있어 **사실상 경로가 하나뿐**이다.

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 6과목 가로 탭. 라벨은 **짧게 ("1과목"~"6과목")** — 풀네임 미사용 |
| 2 | 현재 과목 **활성 하이라이트** |
| 3 | 배치: **과목 목차 페이지 + 챕터 상세 페이지**의 브레드크럼 **아래** 공통 삽입 |
| 4 | 모바일: **가로 스크롤 탭**, 현재 과목이 **처음부터 뷰포트 안에 보이도록** 초기 스크롤 위치 조정 |
| 5 | 데스크톱: 6개 전체 노출 |

**예외사항**
- 모바일에서 6개 탭이 화면에 다 안 들어감 → 가로 스크롤 + 현재 과목 자동 포커싱으로 해결

**화면 흐름**
```
챕터 상세 (4과목) → 탭 바에서 "2과목" 클릭 → 2과목 목차 페이지
```

**정책**
- 탭 클릭 시 이동 대상: ⚠️ **과목 목차 vs 해당 과목 첫 챕터 — 미정의**

**우선순위**: P1
**현재 구현 여부**: 🟡 **Claude Design 지시문 작성 완료. 실제 구현 여부 미확인**

**앞으로 개선 사항**
- 탭 클릭 시 이동 대상 정책 확정
- 실기 추가 시 "필기/실기" 상위 탭이 하나 더 필요해짐 → 2단 탭 구조 검토

## F-15. 챕터 콘텐츠 (5섹션 구조 / v3 템플릿)

**기능명**: 챕터 콘텐츠 템플릿 v3

**목적**
254개 챕터를 **1인이 유지보수 가능한 구조**로 표준화.

**사용자 가치**
모든 챕터가 동일한 구조 → 학습 리듬이 일정. 어디에 무엇이 있는지 예측 가능.

**요구사항**

| 섹션 | 내용 | 생성 방식 |
|------|------|-----------|
| **1. 핵심 공식 / 정의** | 공식(LaTeX) + 정의 + **⚠️ 함정 주의 인용블록** | 수동 |
| **2. 단위·기호 해설** | 기호 / 명칭 / 단위 / 주의점 표 | 수동 |
| **3. 해석 / 의미** | 왜 중요한지, 어디에 응용되는지 | 수동 |
| **4. 기출 출제 이력** | 빈도 라벨 + 회차 뱃지 + 문제 레이어 | **빌드 자동** (수동은 "출제 경향 코멘트" 1~2줄만) |
| **5. 관련 챕터** | 동일 그룹 + 수동 지정 챕터 링크 | **빌드 자동** |

**frontmatter 스키마**
```yaml
title: 옴의 법칙
slug: ohm-law              # 동결 키
subject_id: 4              # 동결 키
group: 기초 전기 공식
tags: [계산공식, 빈출]
summary: 전압·전류·저항의 관계식 V=IR. …
questions: [20220424_061, 20200926_073, …]   # 기출↔챕터 관계의 유일한 저장처
related: []
priority: 출시 필수         # enum: 출시 필수 / 1차 / 2차
status: 작성완료            # enum: 작성완료 / 미작성
order: 12
```

**예외사항**
- **섹션이 없는 챕터**: 해당 섹션 숨김
- **기출 매칭 0건**: ⚠️ 섹션 4 처리 미정의
- **스텁 챕터**: frontmatter만 존재, 본문 빈 상태

**화면 흐름**
해당 없음 (콘텐츠 정의)

**정책**

| 항목 | 값 | 근거 |
|------|-----|------|
| 문체 | **전면 명사형 어미** | 수험 요약의 표준 문체 + 스캔 가독성 |
| 자연 분량 | **1,000~1,300자** (수작업 영역은 700~900자) | 억지로 늘리지 않는다. 초기 "최소 1,500자" 기준은 **폐기** (통합형 챕터 전제였음) |
| 챕터 입자 | **1챕터 = 1개의 독립된 학습 주제** (5~10분 완결) | 너무 잘게 쪼개면 빈약, 너무 크게 묶으면 SEO 비효율 |
| 수식 | **LaTeX + KaTeX 렌더링** | 이미지 대비 로딩·SEO 유리 |
| 기출 링크 | **본문 하드코딩 절대 금지.** frontmatter 배열에 ID만 | 254개 유지보수 불가능해짐 |
| 근사 표기 | **"2018~2022 공통" 같은 뭉뚱그린 표기 금지** | 데이터 서비스의 신뢰도를 깎는다 |

**우선순위**: P0
**현재 구현 여부**: ✅ **74개 작성 완료** (⚠️ 최신 CSV 검증에서는 **작성완료 82개** — 부록 참조)

**앞으로 개선 사항**
- 1차 우선순위 챕터 대량 양산 (스텁 채우기)
- "출제 경향 코멘트"의 품질 편차 관리 — 이것이 유일한 수작업 차별화 지점
- 기출 0건 챕터 처리 정책 확정

## F-16. 기출 출제 이력 뱃지 ★ 차별화 핵심

**기능명**: 기출 출제 이력 (챕터 섹션 4)

**목적**
**"이 개념이 실제로 몇 번, 어떤 회차에 나왔는가"를 데이터로 증명한다.**

**사용자 가치**
- 수험서가 소실시킨 **출제 빈도 정보**를 복원
- "이 챕터 공부할 가치가 있나?"에 대한 즉답
- **경쟁 서비스가 복제할 수 없는 정보** (기출 전수 DB 보유가 전제)

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | **출제 빈도 라벨**: "5개년 N회 출제" — `questions` 배열 길이로 빌드 타임 자동 산출 |
| 2 | **회차 뱃지 목록**: 회차별 개별 표시 (예: "2022년 4월 시행") |
| 3 | **말줄임 / 펼치기**: 기본은 최근 N개만, 나머지는 접힘 |
| 4 | 뱃지 클릭 → **기출문제 모달** (F-17) |
| 5 | **출제 경향 코멘트 1~2줄**: 챕터당 유일한 수작업 영역 |

**예외사항**
- **고빈출 챕터는 관련 문제가 수십 개** → 전량 표시 불가. 말줄임/펼치기가 필수
- **기출 0건 챕터**: ⚠️ 처리 미정의

**화면 흐름**
```
챕터 상세 섹션 4
 ├─ 빈도 라벨 ("5개년 4회 출제")
 ├─ 회차 뱃지 [2022년 4월] [2020년 9월] [2021년 3월] … [+3 더보기]
 │    └─ 뱃지 클릭 → 기출문제 모달 (F-17)
 └─ 출제 경향 코멘트 (수동 작성)
```

**정책**
- 데이터 소스: **챕터 frontmatter의 `questions` 배열** ↔ `questions.json` **빌드 타임 조인**
- 관계의 유일한 저장처는 챕터 쪽. **`questions.json`에는 챕터 정보를 넣지 않는다** (관계 이중화 금지)
- 근사·뭉뚱그린 표기 금지

**우선순위**: P0
**현재 구현 여부**: ✅ 구현 완료

**앞으로 개선 사항**
- 빈도 라벨을 시각화(막대/히트맵)로 강화 검토
- "최근 3년 상승세" 같은 트렌드 파생 지표 (데이터가 이미 있으므로 무상 확보 가능)
- 과목/그룹 단위 출제 빈도 집계 페이지 (SEO 신규 랜딩 페이지가 됨)

## F-17. 기출문제 모달 (문제 레이어)

**기능명**: 기출문제 모달

**목적**
챕터에서 이탈하지 않고 실제 출제 문제를 확인.

**사용자 가치**
"이 개념이 이렇게 나왔구나"를 즉시 확인 → 학습 목표가 구체화됨.

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | 트리거: 챕터 섹션 4의 회차 뱃지 클릭 |
| 2 | 구성: **회차·번호 / 문제 본문 / 선택지 ①~④ / 정답 / 관련 챕터 이동 링크** |
| 3 | 관련 챕터 클릭 → 챕터 페이지로 **Soft Navigation** |
| 4 | 닫기: **X 버튼 / ESC 키 / 모달 밖 클릭** |

**예외사항**
- **이미지가 포함된 문제**: `검수 상태 = "jpg 확필"`로 표시된 문제들. 이미지 처리 방식 ⚠️ **미정의** (현재 텍스트만 표시되어 문제 이해가 불가능할 수 있음)
- 선택지가 빈 문제: 빈 문자열(`""`)로 처리
- 정답이 파싱 불가한 문제: **DB에서 제외**됨 (애초에 노출되지 않음)

**화면 흐름**
```
챕터 상세 → 섹션 4 회차 뱃지 클릭 → 모달 오버레이 열림
 ├─ 문제/선택지/정답 확인
 ├─ "도수율 챕터로 이동" 클릭 → 챕터 페이지 (모달 닫힘)
```

**화면 흐름**
```
어느 페이지든 → GNB 검색 아이콘 (또는 홈 Hero 검색창) → 모달 열림
 ├─ 타이핑 → 실시간 결과
 │    └─ 결과 카드 클릭 → 챕터 상세 (모달 닫힘)

 └─ X / ESC / 밖 클릭 → 모달 닫힘, 원래 페이지 유지
```

**정책**

| 항목 | 값 |
|------|-----|
| `/search` 별도 페이지 | **폐기** (§9 X-02). 모달로 흡수 |
| 광고 | **미배치** |
| SEO | 모달은 SEO 기여 없음 — 감수 |
| 결과 공유 | 불가 (URL 변경 없음) — 감수 |

**우선순위**: P1
**현재 구현 여부**: 🟡 **검색 기술 스택 및 구현 완료 여부 미확인**

> ⚠️ **확실하지 않음 — 검색 엔진**: 정책서에는 **Pagefind**로 확정 기록. 그러나 다른 시점에는 **Lunr.js / Fuse.js**도 후보로 언급됐다. 최종 채택 기술은 `package.json` 확인 필요.

**앞으로 개선 사항**
- **결과 다수 처리 정책 확정** (보류 상태) — 실제 UI를 본 뒤 하이브리드(`/search` 페이지) 도입 여부 결정
- 검색 인덱스에서 스텁 챕터 제외 검증
- 오타 허용(fuzzy) 검색 지원 여부 검토 — 한글 수험 용어 특성상 유용

(직접적 가치 없음 — 무료 서비스를 유지하는 대가)

**요구사항**

| 페이지 | 광고 수 | 위치 |
|--------|---------|------|
| 홈 | 1~2 ⚠️ | 6과목 카드 ↔ 인기 챕터 (+ 인기 ↔ 최근) |

| 필기 페이지 | 1 | 6과목 카드 ↔ 인기 챕터 |
| 과목 목차 | 2 | 그룹 1↔2 사이 + 페이지 하단 |
| **챕터 상세** | **3** | **섹션 3↔5 사이 + 페이지 하단** |

| 검색 모달 | **0** | — |
| `/terms`, `/privacy` | **0** | — |

**예외사항**
- **AdSense 승인 전**: 코드 삽입 위치만 사전 설계하고, **승인 후 활성화**
- 광고 로드 실패 시 레이아웃이 무너지지 않도록 슬롯 높이 예약 (CLS 방지) ⚠️ 구현 확인 필요

**화면 흐름**
해당 없음 (페이지 내 삽입)

**정책**
- **수익의 80%+ 는 챕터 상세 페이지에서 발생 예상.** 그래서 챕터 페이지에만 3개를 배치했다
- **트래픽이 낮은 페이지(자격증 허브)에는 광고를 넣지 않는다.** UX만 해치고 수익 기여는 미미
- **AdSense 승인 요건은 기능 다양성이 아니라 콘텐츠 충실도.** 이것이 MVP에서 기능을 계속 잘라낸 근거

**우선순위**: P0
**현재 구현 여부**: 🟡 **배치 설계 확정. AdSense 미신청 상태이므로 실제 광고는 미노출**

**앞으로 개선 사항**
- **AdSense 신청** (선행: 콘텐츠 충실도 + 정책 페이지 + 커스텀 도메인 연결)
- 승인 후 광고 위치별 RPM 측정 → 재배치
- 광고 밀도가 UX를 해치는지 이탈률로 검증

## F-33. `/admin/` 현황 페이지

**기능명**: 챕터 관리 현황 페이지

**목적**
**노션을 대체하는 "자동 생성 장부".** 254개 챕터의 작성 현황을 한눈에 조회.

**사용자 가치 (운영자)**
- "미작성 몇 개 남았나"를 즉시 확인
- **노션과 달리 실물(md 파일)에서 자동 생성되므로 어긋날 방법이 물리적으로 없다**

**요구사항**

| # | 요구사항 |
|---|----------|
| 1 | `src/pages/admin/index.astro` — 전체 챕터(**스텁 포함**)를 `getCollection`으로 읽어 표로 나열 |
| 2 | 컬럼: **챕터명**(작성완료면 링크) / 과목 / 그룹 / 우선순위 / 상태 / **기출 매칭 수**(`questions` 배열 길이) |
| 3 | 상단 요약 1줄: 전체 N / 작성완료 N / 미작성 N (우선순위별 괄호 병기) |
| 4 | 정렬: 과목 → `order` |
| 5 | 스타일: **기존 `global.css` 토큰·표 스타일 재사용. 신규 디자인 금지** |
| 6 | `<meta name="robots" content="noindex, nofollow">` |
| 7 | `robots.txt`에 `Disallow: /admin/` |
| 8 | sitemap에서 제외: `sitemap({ filter: (page) => !page.includes('/admin') })` |

**예외사항**
- **인증 없음.** URL을 아는 사람은 누구나 접근 가능. 민감 정보가 없으므로 감수 (noindex로 검색 노출만 차단)
- **조회 전용. 편집 불가** — 이 한계를 명시적으로 인지하고 수용했다

**화면 흐름**
```
직접 URL 입력 (/admin/) → 254행 테이블 조회
 └─ 작성완료 챕터명 클릭 → 해당 챕터 페이지
```

**정책**
- **노션 표 = 사람이 갱신하는 장부 / admin = 실물에서 자동 생성되는 장부.** 후자는 어긋날 수 없다
- **한계 수용 근거**: 마일로의 워크플로가 이미 "**편집은 지시, 확인만 직접**"이므로 조회 전용이어도 실질 손실이 없다
- 유지 비용 0 (정적 페이지, 서버·DB 없음)
- 백업·이력은 git 히스토리가 공짜로 제공

**우선순위**: P0 (운영 필수)
**현재 구현 여부**: 🟡 **이관 지시문 PART 5에 포함. 실행 완료 여부 미확인**

**앞으로 개선 사항**
- 정렬 가능한 테이블 (클라이언트 사이드 소팅)
- 본문 파생 지표 추가: **글자 수, 수식 포함 여부** (본문을 DB에 넣지 않고도 집계 가능)
- 기출 매칭 0건 챕터 하이라이트 (콘텐츠 우선순위 재조정에 유용)

## F-34. 챕터 스텁 시스템 (status 빌드 필터)

**기능명**: 스텁 챕터 + 빌드 필터

**목적**

```
문제 풀이 / 기출 모달 → 오답 발생 → 자동 저장 → 오답노트 페이지에서 재학습
```

**우선순위**: P3
**현재 구현 여부**: ⬜ 미구현

**앞으로 개선 사항**

**요구사항**

**정책**

- 정적 사이트에서도 풀이·채점은 가능 (기록 저장만 DB 필요)

**우선순위**: P3
**현재 구현 여부**: ⬜ 미구현 (기출 모달로 부분 대체 중)

**앞으로 개선 사항**

- 비회원 제한 정책 확정

## F-45. 구독 결제 (Stripe)

**기능명**: 구독 결제

**목적**
반복 수익 + **포트폴리오의 결제 시스템 구현 경험**.

**사용자 가치**
광고 없이, 개인화된 학습 도구를 이용.

**요구사항**

```
사용자 회원가입 (Supabase Auth)
    ↓
결제 페이지 진입 (Stripe Checkout)
    ↓
결제 완료 (Webhook → Supabase에 구독 상태 저장)
    ↓
프리미엄 기능 잠금 해제 (로그인 시 subscription 상태 확인)
```

- Stripe Checkout
- Webhook으로 구독 상태 → Supabase 동기화
- 테스트 모드로 결제 플로우 선검증

**예외사항**
- 구독 만료/취소/결제 실패 시 기능 접근 회수 로직 필요
- 환불 정책 ⚠️ 미정의

**화면 흐름**
```
프리미엄 기능 시도 → 페이월 → Stripe Checkout → 결제 → 기능 해제
```

**정책**

| 항목 | 상태 |
|------|------|
| **가격 모델 (월간/연간/둘 다)** | ⚠️ **미결정** |
| **가격대** | ⚠️ **미결정** (월 4,900원 / 9,900원 후보 논의만) |
| **유료 기능 경계선** | ⚠️ **미결정** |

- **스택 선택 근거**: Stripe은 실리콘밸리 B2C 스탠다드 → **포트폴리오 신뢰도**. Supabase-Stripe 공식 통합 지원. 반복 청구 기본 지원
- **대안 검토**: Firebase+Stripe(수동 연동 필요), Toss Payments+Supabase(국내 결제 — 검토됨)

**우선순위**: P4
**현재 구현 여부**: ⬜ 미구현

**앞으로 개선 사항**
- **가격·유료 경계선 결정 (Phase 2 진입 전 필수)**
- 국내 사용자 대상이므로 **Toss Payments 재검토** 여지 있음 (Stripe는 국내 카드 결제 지원 이슈가 있을 수 있음 ⚠️ **최신 확인 필요**)

## F-47. 모의고사 (기출 랜덤 출제)

**기능명**: 모의고사

**목적**
실전 감각 훈련.

**사용자 가치**
실제 시험과 동일한 구성(6과목 × 20문제 = 120문제)으로 연습.

**요구사항**
- **A타입: 기출문제 랜덤 출제** (AI 없음, 정적으로도 구현 가능)
- 채점 및 합격 여부 판정 (평균 60점 이상, 과목별 40점 이상)

**예외사항**
- 재시도 횟수 제한: 회원(무료)은 기록 저장, 유료는 **무제한 재도전**

**화면 흐름**
```
모의고사 시작 → 120문제 풀이 → 채점 → 결과 리포트 → 오답 저장
```

**정책**
- **"AI 기반 모의고사"라는 표현이 있었으나, 실제 정의는 A타입(기출 랜덤 출제) + B타입(오답 기반 추천)이다.** **AI가 새 문제를 생성하는 C타입은 채택하지 않았다** (마일로 확인)

**우선순위**: P4
**현재 구현 여부**: ⬜ 미구현

**앞으로 개선 사항**
- 실제로는 **정적 사이트에서도 구현 가능**하다 (문제 랜덤 추출 + 클라이언트 채점). Phase 1 후반에 앞당길 여지 있음 — **트래픽·체류시간 증대 효과가 클 수 있으므로 재검토 권장**

## F-48. 약점 문제 추천

**기능명**: 약점 문제 추천

**목적**
오답 데이터를 기반으로 틀리는 유형을 반복 노출.

**사용자 가치**
공부 시간 대비 합격률 향상.

**요구사항**
- **B타입: 사용자 오답 기반 약점 문제 추천** (DB + 간단한 로직. AI 아님)
- `attempts`, `wrong_notes` 테이블 기반

**예외사항**
- 오답 데이터가 적으면 추천 정확도가 낮음 → 최소 시도 횟수 임계값 필요

**화면 흐름**
```
문제 풀이 누적 → 약점 분석 → "이 유형 다시 풀기" 추천
```

**정책**
- **유료 구독 기능**
- AI가 아니라 **DB 조회 + 로직**이다. (마일로가 A·B 타입으로 확정)

**우선순위**: P4
**현재 구현 여부**: ⬜ 미구현

**앞으로 개선 사항**
- 챕터-기출 매핑(F-36)을 역이용: 오답 문제 → 연결된 챕터 → **"이 챕터를 다시 보세요"** 추천. **데이터가 이미 있으므로 저비용으로 구현 가능**

# §9. 폐기된 기능 · 아이디어

> **왜 이 섹션이 존재하는가**: 폐기 사유를 남기지 않으면 6개월 뒤에 같은 아이디어가 다시 올라온다. 각 항목은 **재검토 조건**까지 명시한다.

## X-01. 빈출 챕터 미리보기 카드 ❌

**내용**: 홈의 6과목 카드에 "▶ 빈출: 재해 비율 법칙" 한 줄을 추가하는 안.

**폐기 사유**
- MVP 범위에서 운영 부담(선정 기준 / 갱신 주기 / 과목당 1개씩 6개 선정) 대비 효용 불명확
- 마일로 판단: *"MVP에서는 빠져도 될 것 같음. 아니면 나중에 추가해도 되고"*

**검토됐던 3안**: A(포함+고정, 마일로 1회 선정) / B(포함+동적, 인기 챕터 자동) / C(제외) → **C 채택**

**재검토 조건**: 인기 챕터 데이터가 GA4로 자동 산출되기 시작하면 B안(동적)의 운영 부담이 0이 됨 → 그때 재검토 가치 있음

---

## X-02. `/search` 별도 검색 페이지 ❌ (조건부 보류)

**내용**: 사이트 전체 검색 결과를 보여주는 독립 페이지.

**폐기 사유**
- 검색 모달로 흡수. 별도 페이지를 만들 필요가 없다
- 모달은 사용자 현재 위치를 유지 → 빠른 검색 → 빠른 닫기

**감수한 단점**: URL이 안 바뀜 → 검색 결과 공유 불가. SEO 페이지 1개 안 늘어남 (영향 미미)

**재검토 조건**: **하이브리드안(모달 Top 5 → "전체 결과 25건 보기" → `/search` 페이지)이 제안되었고, 마일로가 "UI를 봐야 결정"이라며 보류했다.** 검색 결과가 많을 때의 가독성 문제가 실제로 확인되면 부활 대상.

---

## X-03. 챕터 7섹션 구조 ❌

**내용**: 초기 챕터 템플릿은 7섹션이었다.
`1.핵심공식 / 2.단위·기호 / 3.해석 / 4.예시 계산 / 5.기출 출제 패턴 / 6.함정·자주 틀리는 부분 / 7.관련 챕터`

**폐기 사유**
- **254개 양산이 불가능한 수작업량**
- 5섹션으로 축소하면서: **"예시 계산"과 "함정" 섹션이 제거**됨. 함정 내용은 **섹션 1의 인용블록(⚠️ 함정 주의)으로 흡수**
- 섹션 4·5를 **빌드 타임 자동 생성**으로 전환 → 수작업이 700~900자로 급감

**부수 폐기**: "핵심 섹션 1·2·3 펼침 + 4~7 접기" 정책도 함께 폐기. 5섹션 체제에서는 **전부 펼침**이 맞고, 접기는 SEO에도 불리하다.

**재검토 조건**: 없음. 이 결정 덕에 74개 양산이 실제로 완료됐다.
