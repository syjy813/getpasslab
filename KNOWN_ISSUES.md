## ISS-01. `status` 빌드 필터 부재 🟢

| 속성 | 값 |
|------|-----|
| **카테고리** | 버그 (데이터 노출) |
| **심각도** | 🔴 P0 |
| **상태** | ⏳ 대기 (이관 세션에서 처리 예정) |
| **관련** | F-34, ROADMAP B-3, CODING_RULES §8.2 |

**문제**
스텁 챕터(미작성)를 걸러내는 `status === '작성완료'` 필터가 **현재 코드에 없다.**

```ts
// 실제 코드 (written/index.astro) 🟢
const countBy = (id: number) => chapters.filter(c => c.data.subject_id === id).length;
//                                                    ^^^ status 필터 없음
```

**영향**
- 스텁 171개를 생성하면 → **"챕터 48개"라고 표시하고 클릭하면 12개만 나온다**
- `getStaticPaths`에도 필터가 없으면 **빈 스텁 페이지가 배포된다** → SEO에 빈 페이지 노출
- 사이드바에 미작성 챕터가 뜨면 → **클릭 시 404**

**재현 조건**
`B-2(스텁 생성)`를 실행하고 필터를 안 넣으면 즉시 발생.

**해결**
필터 4곳 추가:

| # | 위치 | 필터 |
|---|------|------|
| 1 | 챕터 상세 `getStaticPaths` | `.filter(c => c.data.status === '작성완료')` |
| 2 | 과목 목차 카드·카운트 | 동일 |
| 3 | 홈 과목 카드 카운트 | 동일 |
| 4 | 챕터 사이드바 `sideChapters` | 동일 |

**🚨 스텁 생성(B-2)과 필터 추가(B-3)는 반드시 같은 커밋에 들어가야 한다.**

**검증**
```bash
npm run build
find dist -name index.html | wc -l   # 작성완료 챕터 수와 일치해야 함
# dist에 미작성 챕터 페이지가 있으면 → 실패
```

**문제**
도형·회로도 등 이미지가 포함된 기출문제는 노션에 `검수 상태 = "jpg 확필"`로 **표기만** 되어 있고, **이미지 자체가 없다.**

**영향**

```bash
python3 -c "
import json; qs=json.load(open('src/data/questions.json'))
img=[q for q in qs if q.get('review')]
print(f'이미지 문제: {len(img)}건 / 전체 {len(qs)}건')
"
```
**이 숫자를 모르면 의사결정을 할 수 없다.**

**해결 (단계적)**

| 안 | 내용 | 난이도 | 권장 |
|----|------|--------|------|
| **A** | 해당 문제를 모달에서 **노출 제외** | 🟢 | **1차 방어 (즉시)** |

| **C** | "이미지 문제 — 원본 참조" 배지 | 🟢 | 비권장 |

> ⚠️ **B의 저작권**: 기출 이미지는 공공 저작물 범주일 가능성이 높으나, **해설집 PDF의 재가공 이미지라면 다르다.** 확인 필요.

## ISS-04. `filter(Boolean)` 조용한 실패 🟢

| 속성 | 값 |
|------|-----|
| **카테고리** | 버그 (데이터 무결성) |
| **심각도** | 🔴 P0 |
| **상태** | 🆕 미착수 |
| **관련** | F-16, F-36, CODING_RULES §5.4, DECISION D-73 |

**문제**
챕터 frontmatter의 `questions` 배열에 **오타/잘못된 문제 ID가 있어도 빌드가 통과하고, 해당 기출 뱃지만 조용히 누락**된다.

```ts
// 실제 코드 🟢
const linked = chapter.data.questions
  .map(id => allQuestions.find(q => q.data.id === id)?.data)
  .filter(Boolean)   // ← 존재하지 않는 ID가 여기서 조용히 사라진다
  .sort((a, b) => b!.id.localeCompare(a!.id));
```

**영향**
- 253개 챕터 중 **어디서 누락됐는지 알 방법이 없다**
- 🚨 **노션 relation을 대량 병합하는 이관 세션(B-2) 직후에 특히 위험** — 병합 과정에서 오타 ID가 섞일 가능성이 실재

**해결 — 미매칭 ID 경고 출력 + 타입 가드**
```ts
const qMap = new Map(allQuestions.map(q => [q.data.id, q.data]));  // ISS-17도 해결
const missing: string[] = [];
const linked = chapter.data.questions
  .map(id => { const q = qMap.get(id); if (!q) missing.push(id); return q; })
  .filter((q): q is NonNullable<typeof q> => q !== undefined)      // 타입 가드 → ! 불필요
  .sort((a, b) => b.id.localeCompare(a.id));

if (missing.length) console.warn(`⚠️ [${chapter.data.slug}] 미매칭 문제 ID: ${missing.join(', ')}`);
```

> **B-2 실행 전에 넣어야 오류를 잡는다.** 순서가 중요하다.

## ISS-05. BaseLayout Pretendard 링크 누락 🟢

| 속성 | 값 |
|------|-----|
| **카테고리** | 버그 (렌더링) |
| **심각도** | 🔴 P0 |
| **상태** | ❓ 확인 필요 |
| **관련** | F-13, DECISION D-28, CODING_RULES §9.5 |

**문제**
Pretendard 폰트 링크가 `ChapterLayout.astro`에만 있고 `BaseLayout.astro`에는 없다.

```astro
<!-- BaseLayout.astro 🟢 — KaTeX만 -->
<link rel="stylesheet" href="…katex.min.css" />

<!-- ChapterLayout.astro 🟢 — Pretendard 있음 -->
<link rel="stylesheet" href="…pretendardvariable…" />
```

**영향**
BaseLayout을 쓰는 **홈·자격증 허브·필기·과목 목차·admin 페이지에서 Pretendard가 로드되지 않아 시스템 폰트로 폴백**될 수 있다. 첫인상이 깨진다.

**확인 먼저**
```bash
grep -n "@import" src/styles/global.css   # global.css에서 폰트를 import하면 문제 없음
grep -rn "pretendard" src/layouts/
```
**`global.css`에 `@import`가 있으면 문제 없음. 없으면 버그.**

**해결**: BaseLayout `<head>`에 Pretendard 링크 추가, 또는 `global.css`에 `@import` 추가. → **SEO 메타 리팩터링(ISS-12)과 함께 `<SEO>` 컴포넌트로 통합하면 한 번에 해결.**

**영향**

| B | 완전 실명 공개 | — |
| C | 이메일만, 실명 생략 | 법적 요건 미달 가능 ⚠️ |

> ⚠️ **확실하지 않음**: 한국 개인정보보호법상 개인 운영 사이트의 책임자 표기 의무 범위, AdSense의 정확한 요구 수준. **신청 전 최신 기준 확인 필요.**

## ISS-17. 조인 O(n×m) 복잡도 🟢

| 카테고리 | 성능 · 심각도 🟡 P2 · 상태 🆕 |

```ts
chapter.data.questions.map(id => allQuestions.find(q => q.data.id === id))  // O(n×m)
```

**영향**: 지금은 빌드가 빠르지만, **자격증 확장으로 문항이 5,000개+ 되면 빌드 시간이 눈에 띄게 증가.**

```ts
const qMap = new Map(allQuestions.map(q => [q.data.id, q.data]));
chapter.data.questions.map(id => qMap.get(id))  // O(n)
```
